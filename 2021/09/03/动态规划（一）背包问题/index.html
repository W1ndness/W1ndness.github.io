前言

本文将介绍动态规划中的背包问题。将介绍四种背包问题，并给出其经典的优化方式，最后给出算法模板。另外，本文中还会给出 DP 问题的分析方式（闫式dp）

动态规划（一）背包问题

〇、什么是背包问题，如何从dp分析背包问题

一、01 背包问题

1.1 01 背包问题描述

要求为每件物品 iii​​​ 只能使用一次，即 ∀αi∈N, si=1\forall \alpha_i\in N,\ s_i=1∀αi​∈N, si​=1。

则数学描述为：

\begin{align} \text{get }\alpha_i\in N\Rightarrow\alpha=(\alpha_1,\alpha_2,\cdots,\alpha_m),\ \forall \alpha_i\in N,\ s_i=1\\[3px] \mathrm{s.t.} \ \ \max\sum_\alpha w_i\text{ , where } \sum_\alpha v_i\le V \end{align}

1.2 01 背包的 dp 分析

  • 状态表示 f(i,j)f(i,j)f(i,j)

    • 集合：该状态表示的是哪一个集合
    • 表示所有选法的集合
    • 满足的条件
      • 只能从前 iii 个物品里选
      • ∑αvi≤j\sum_\alpha v_i\leq j∑α​vi​≤j​​
    • 属性：需要存储的是什么属性
      • 最大值：价值最大

  • 状态计算 —— 集合的划分：不重、不漏

    对于 01 背包问题，集合的划分方式为：f(i,j)⇒{f(i,j,N1),f(i,j,N2)}f(i,j)\Rightarrow\{f(i,j,N_1),f(i,j,N_2)\}f(i,j)⇒{f(i,j,N1​),f(i,j,N2​)}​​​ 。其中， i∉N1,i∈N2i\notin N_1,i\in N_2i∈/​N1​,i∈N2​​​​ ，N1,N2⊂NN_1,N_2\subset NN1​,N2​⊂N​​​​​​​​ 。

    故得到初步的状态转移方程为

    f(i,j)=max⁡{f(i,j,N1),f(i,j,N2)}f(i,j)=\max\{f(i,j,N_1),f(i,j,N_2)\} f(i,j)=max{f(i,j,N1​),f(i,j,N2​)}

    由于状态表示中对状态的条件定义，我们又能将集合划分后的三维状态进行等价降维，即：

    \begin{align} f(i,j)&=\max\{f(i,j,N_1),f(i,j,N_2)\}\\ &\iff\max\{f(i-1,j),f(i-1,j-v_i)+w_i\big\} \end{align}

    证明：

    1. 对于不包含物品 iii​​ 的情况 f(i,j,N1), i∉N1⊂Nf(i,j,N_1),\ i\notin N_1\subset Nf(i,j,N1​), i∈/​N1​⊂N

      我们要求状态 f(x,y)f(x,y)f(x,y)​ 表示只能从前 xxx​ 个物品中选取体积不超过 yyy​ 的物品，那么如果选取的集合要从前 iii​ 个物品里选，但我们还要求不包含 iii​ ，那么也就相当于从前 i−1i-1i−1 个物品中进行选取，故原三维状态可降维至二维状态 f(i−1,j)f(i-1,j)f(i−1,j)​ 。

    2. 对于包含物品 iii​ 的情况 f(i,j,N2), i∈N2⊂Nf(i,j,N_2),\ i\in N_2\subset Nf(i,j,N2​), i∈N2​⊂N

      这里稍有些难以理解，我们要从前 iii​​ 个中选取物品，还要包含第 iii​​ 个物品，并且体积限制在 ≤j\leq j≤j​​ 。那么可以看到，这些选取方式是不是都选择了物品 iii​​​ ？那么好了，逆向思维想一下，既然它一定会被选取，我们也就可以不去看这个物品 iii​​ ，由于对于状态的条件定义，我们这时就是从前 i−1i-1i−1​​ 个物品中进行选取，由于物品 iii​​ 一定会被选到，我们现在还没有将其算入我们的 “选定范围” 以内，那么体积的上限也就减小至 ≤j−vi\leq j-v_i≤j−vi​​​​ ，那么这个对应的状态表示也就是 f(i−1,j−vi)f(i-1,j-v_i)f(i−1,j−vi​)​ 。由于我们前面忽视了物品 iii​ ，又因为这一集合包含的所有选法中是必然含有物品 iii​ 的，所以我们最后需要把这一物品的价值加上才得到了包含物品 iii​ 的完整选取方式。故原三维状态可降维为 f(i−1,j−vi)+wif(i-1,j-v_i)+w_if(i−1,j−vi​)+wi​​​​ 。

    证毕。

    注：等价降维的思想就是找到状态表示定义下某状态的意义，并找到与其等价的某一状态或去掉某一元素后等价的某一状态，即可进行等价降维。

    那么我们得到了如何计算状态，即 01 背包问题状态转移方程：

    f(i,j)=max⁡{f(i−1,j),f(i−1,j−vi)+wi}f(i,j)=\max\{f(i-1,j),f(i-1,j-v_i)+w_i\} f(i,j)=max{f(i−1,j),f(i−1,j−vi​)+wi​}

二维的01背包

int n, m;
int v[N], w[N];
int f[N][N];

int main() {
    /* read in */
    /* f[0][0]~f[0][m] = 0 */
    for (int i = 1; i <= n; i++) {
        for (int j = 0; j <= m; j++) {
            f[i][j] = f[i - 1][j];
            if (j >= v[i]) f[i][j] = max(f[i][j], f[i - 1][j - v[i]] + w[i]);
            // 包含i的情况在 总体积不够v[i]时 就是空集，是无效计算
        }
    }
    /* output f[n][m] */
    return 0;
}

1.3 01 背包的一维优化

由于我们的 f(i,⋅)f(i,\cdot)f(i,⋅)​ 只用到了 f(i−1,⋅)f(i-1,\cdot)f(i−1,⋅)​ 的情况，则可以用滚动数组做。而且始终有有 j,j−vi≤jj,j-v_i\leq jj,j−vi​≤j​​​​​ 成立 ，那么状态表示就可以减到一维。

那么原来的状态转移方程就被改成了

f(j)=max⁡{f(j),f(j−vi)+wi}f(j)=\max\{f(j),f(j-v_i)+w_i\} f(j)=max{f(j),f(j−vi​)+wi​}

那么直接降维后的这个状态转移方程是否和二维的方程等价呢？答案是否定的。

引入符号 gk(V)g_k(V)gk​(V)​​​ 为第 kkk​​​ 轮更新的体积 ≤V\leq V≤V​​​ 的状态，即 gx(y)  ⟺  f(x,y)g_x(y)\iff f(x,y)gx​(y)⟺f(x,y) 。前文提到过， j−vi≤jj-v_i\leq jj−vi​≤j​​​​​ 始终成立。由于我们的循环是从小到大进行枚举的，那么实际上 f(j−vi)f(j-v_i)f(j−vi​) 一定是在 f(j)f(j)f(j) 之前被更新的。那现在的 f(j−vi)f(j-v_i)f(j−vi​) 其实应该是 f(i,j−vi)f(i,j-v_i)f(i,j−vi​) ，或者写做 gi(j−vi)g_i(j-v_i)gi​(j−vi​)​ 。那么现在的状态转移方程实质上是：

\begin{align} g_i(j)&=\max\{g_i(j),g_i(j-v_i)+w_i\}\\ &\iff \max\{f(i,j),f(i,j-v_i)+w_i\} \end{align}

可以看到，这在 max⁡\maxmax 算子中的第二项是完全不同的。那么应该如何修改呢？我们应让 g⋅(j−vi)g_\cdot(j-v_i)g⋅​(j−vi​) 这一项是 gi−1(j−vi)g_{i-1}(j-v_i)gi−1​(j−vi​) 而不是 gi(j−vi)g_i(j-v_i)gi​(j−vi​) 。还是由于 j−vi≤jj-v_i\leq jj−vi​≤j 严格成立，我们只需先更新 g⋅(j)g_\cdot(j)g⋅​(j) ，后更新 g⋅(j−vi)g_\cdot(j-v_i)g⋅​(j−vi​)​ 就可以了。那么严谨的一维的状态转移方程应被写作：

gi(j)=max⁡{gi−1(j),gi−1(j−vi)+wi}g_i(j)=\max\{g_{i-1}(j),g_{i-1}(j-v_i)+w_i\} gi​(j)=max{gi−1​(j),gi−1​(j−vi​)+wi​}

int n, m;
int v[N], w[N];
int f[N];

int main() {
    /* read in */
    /* f[0][0]~f[0][m] = 0 */
    for (int i = 1; i <= n; i++) {
        for (int j = m; j >= v[i]; j--) {	
            // 循环倒序更新使得 f[j-v[i]] 后被更新，先使用的是之前一轮的 f[j-v[i]] 值
            f[j] = max(f[j], f[j - v[i]] + w[i]);
        }
    }
    /* output f[m] */
    return 0;
}

二、完全背包问题

2.1 完全背包问题描述

要求为每件物品 iii​​​ 有无限个，即 ∀αi∈N, si=+∞\forall \alpha_i\in N,\ s_i=+\infty∀αi​∈N, si​=+∞​​​。

则数学描述为：

\begin{align} \text{get }\alpha_i\in N\Rightarrow\alpha=(\alpha_1,\alpha_2,\cdots,\alpha_m),\ \forall \alpha_i\in N,\ s_i=+\infty\\[3px] \mathrm{s.t.}\ \ \ \ \max\sum_\alpha w_i\text{ , where } \sum_\alpha v_i\le V \end{align}

2.2 完全背包的 dp 分析

  • 状态表示 f(i,j)f(i,j)f(i,j)

    • 集合：该状态表示的是哪一个集合
      • 表示所有选法的集合
      • 满足的条件
        • 只能从前 iii 个物品里选
        • ∑αvi≤j\sum_\alpha v_i\leq j∑α​vi​≤j​​
    • 属性：需要存储的是什么属性
      • 最大值：价值最大

  • 状态计算 —— 集合的划分：不重、不漏

    对于完全背包问题，集合的划分方式为 f(i,j)⇒{f(i,j,0),f(i,j,1),f(i,j,2),⋯ ,f(i,j,m)}f(i,j)\Rightarrow\{f(i,j,0),f(i,j,1),f(i,j,2),\cdots,f(i,j,m)\}f(i,j)⇒{f(i,j,0),f(i,j,1),f(i,j,2),⋯,f(i,j,m)}​​​​ ，其中第三个值代表选取 iii​​​​ 的个数，并假设最多可以取得 mmm​​​ 个，即满足 m=arg⁡max⁡x for αi∑αvim=\displaystyle\mathop{\arg\max}_{x\text{ for }\alpha_i} \sum_\alpha v_im=argmaxx for αi​​α∑​vi​​​。

    下面来看一下每个元素的值是什么：

    \begin{gather} f(i,j,0)=f(i-1,j)\\ \vdots\\ f(i,j,k)=f(i-1,j-kv_i)+kw_i\\ \vdots\\ f(i,j,m)=f(i-1,j-mv_i)+mw_i \end{gather}

    推导的思路和上面 01 背包的完全相同，也是逆向思维曲线救国，这里不再赘述。

    于是得到了完全背包的状态转移方程

    f(i,j)=max⁡{f(i−1,j),f(i−1,j−vi)+wi,f(i−1,j−2vi)+2wi,⋯ ,f(i−1,j−mvi)+mwi}f(i,j)=\max\{f(i-1,j),f(i-1,j-v_i)+w_i,f(i-1,j-2v_i)+2w_i,\cdots,f(i-1,j-mv_i)+mw_i\} f(i,j)=max{f(i−1,j),f(i−1,j−vi​)+wi​,f(i−1,j−2vi​)+2wi​,⋯,f(i−1,j−mvi​)+mwi​}

    这里简写做：

    f(i,j)=max⁡{f(i−1,j),f(i−1,j−kvi)+kwi} , k∈[1,m]⊂Zf(i,j)=\max\{f(i-1,j),f(i-1,j-kv_i)+kw_i\}\ ,\ k\in[1,m]\subset\mathbb Z f(i,j)=max{f(i−1,j),f(i−1,j−kvi​)+kwi​} , k∈[1,m]⊂Z

    进一步简写为：

    f(i,j)=max⁡{f(i−1,j−kvi)+kwi} , k∈[0,m]⊂Zf(i,j)=\max\{f(i-1,j-kv_i)+kw_i\}\ ,\ k\in[0,m]\subset\mathbb Z f(i,j)=max{f(i−1,j−kvi​)+kwi​} , k∈[0,m]⊂Z

    要注意的是实际上严谨的描述这里实际上 kkk 应该能够取到 +∞+\infty+∞​ ，即更准确描述应为

    f(i,j)=max⁡{f(i−1,j−kvi)+kwi} , k∈Z+f(i,j)=\max\{f(i-1,j-kv_i)+kw_i\}\ ,\ k\in\mathbb Z^+ f(i,j)=max{f(i−1,j−kvi​)+kwi​} , k∈Z+

    只不过我们一定存在着最多取得物品 iii​​ 数量的上限，如果溢出上限，选法集合就是空集，那么重复进行 max⁡\maxmax 也没有什么意义。

三维的完全背包

int n, m;
int v[N], w[N];
int f[N][N];

int main() {
    /* read in */
    for (int i = 1; i <= n; i++) {
        for (int j = 0; j <= m; j++) {
            for (int k = 0; k * v[i] <= j; k++) {
                f[i][j] = max(f[i][j], f[i - 1][j - v[i] * k] + w[i] * k);
            }
        }
    }
    /* output f[n][m] */
    return 0;
}

2.3 完全背包的二维优化

可以看到上面的状态转移方程是三维的，与 i,j,ki,j,ki,j,k 都有关系，那么那么如何能够优化至二维呢？

我们有 f(i,j)f(i,j)f(i,j) 的展开为

f(i,j)=max⁡{f(i−1,j),f(i−1,j−vi)+wi,f(i−1,j−2vi)+2wi,f(i−1,j−3vi)+wi,⋯ }f(i,j)=\max\{f(i-1,j),f(i-1,j-v_i)+w_i,f(i-1,j-2v_i)+2w_i,f(i-1,j-3v_i)+w_i,\cdots\} f(i,j)=max{f(i−1,j),f(i−1,j−vi​)+wi​,f(i−1,j−2vi​)+2wi​,f(i−1,j−3vi​)+wi​,⋯}

当 j=j−vij=j-v_ij=j−vi​ ，我们有 f(i,j−vi)f(i,j-v_i)f(i,j−vi​) 的展开为

f(i,j−vi)=max⁡{f(i−1,j−vi),f(i−1,j−2vi)+wi,f(i−1,j−3vi)+2wi,⋯ }f(i,j-v_i)=\max\{f(i-1,j-v_i),f(i-1,j-2v_i)+w_i,f(i-1,j-3v_i)+2w_i,\cdots\} f(i,j−vi​)=max{f(i−1,j−vi​),f(i−1,j−2vi​)+wi​,f(i−1,j−3vi​)+2wi​,⋯}

对比发现，f(i,j−vi)f(i,j-v_i)f(i,j−vi​) 的展开式中 max⁡\maxmax 算子中的部分恰与 f(i,j)f(i,j)f(i,j) 展开式中 max⁡\maxmax 算子内第二项以后的部分近乎相同，只相差了一个 wiw_iwi​ 。由于完全背包问题中 si=+∞s_i=+\inftysi​=+∞ ，两部分是等势的。那么显然，我们可以将 f(i,j)f(i,j)f(i,j) 中第二项以后的部分直接变为 f(i,j−vi)+wif(i,j-v_i)+w_if(i,j−vi​)+wi​​ ，原理上面已经多次提到过了。那么就可以简化其状态转移方程为

f(i,j)=max⁡{f(i−1,j),f(i,j−vi)+wi}f(i,j)=\max\{f(i-1,j),f(i,j-v_i)+w_i\} f(i,j)=max{f(i−1,j),f(i,j−vi​)+wi​}

int n, m;

int main() {

        for (int j = 0; j <= m; j++) {

    f(i,j)=max⁡{f(i−1,j),f(i,j−vi)+wi}f(i,j)=\max\{f(i-1,j),\boxed{f(i,j-v_i)+w_i}\} f(i,j)=max{f(i−1,j),f(i,j−vi​)+wi​​}

    f(i,j)=max⁡{f(i−1,j),f(i−1,j−vi)+wi}f(i,j)=\max\{f(i-1,j),\boxed{f(i-1,j-v_i)+w_i}\} f(i,j)=max{f(i−1,j),f(i−1,j−vi​)+wi​​}

2.4 完全背包的一维优化

3.1 多重背包问题描述

要求为每件物品 iii​​​​ 数量有限，即 ∀αi∈N, si∈Z+\forall \alpha_i\in N,\ s_i\in\Z^+∀αi​∈N, si​∈Z+​​​​。

四、分组背包问题

4.1 分组背包问题描述

要求为共有 \cal N​​​​​​​​ 组物品，要求每一组内只能选取一个物品，记每组物品中物品个数为 \{\cal S _\cal N\}​​ ，物品体积为 \{\cal V_{\cal N\cal S_\cal N}\} ，价值为 \{\cal W_{\cal N\cal S_\cal N}\}。记取出的每个物品所在的组别为 \cal \{G_n\}\subset\cal N ，即存在 \alpha_i\mapsto \cal G_i​​​​​​​​​ 。

则数学描述为：

\begin{align} \text{get }\alpha_i\in N\Rightarrow\alpha=(\alpha_1,\alpha_2,\cdots,\alpha_m),\ \forall \alpha_i\in N,\ s_i\in\mathbb Z^+,\ \forall \alpha_i,\alpha_j,\ \cal G_i\ne\cal G_j\\[3px] \mathrm{s.t.}\ \ \ \ \max\sum_\alpha w_i\text{ , where } \sum_\alpha v_i\le V \end{align}

4.2 分组背包的dp分析

  • 状态表示 f(i,j)f(i,j)f(i,j)

    • 集合：该状态表示的是哪一个集合
      • 表示所有选法的集合
      • 满足的条件
        • 只能从前 iii​ 组物品里选
        • ∑αvi≤j\sum_\alpha v_i\leq j∑α​vi​≤j​​
    • 属性：需要存储的是什么属性
      • 最大值：价值最大

  • 状态计算 —— 集合的划分：不重、不漏

    分组背包问题需要注意，这里的条件变了，是从组中选择物品。那么分组背包问题的集合划分方式为 f(i,j)\Rightarrow\{f(i,j,0),f(i,j,1),\cdots,f(i,j,\cal S_i)\}​​​​ 。这里第三个维度值代表了选取组别 iii 中的第几个元素。这里来分析一下集合中每一个元素的值是什么。

    \begin{gather} f(i,j,0)=f(i-1,j)\\ \vdots\\ f(i,j,k)=f(i-1,j-\cal V_{ik})+\cal W_{ik}\\ \vdots\\ f(i,j,\mathcal{S_i})=f(i-1,j-\mathcal{V_{i\mathcal S_i}})+\cal W_{iS_i}\\ \end{gather}

代码的写法其实有了前面三个背包问题的分析这里也就是大同小异了。由于各种理由（这里略去），分组背包可以被简化至一维，使用滚动数组。

const int N = 110;

int n, m;
int v[N][N], w[N][N], s[N];
int f[N];

int main() {
    cin >> n >> m;
    for (int i = 1; i <= n; i++) {
        cin >> s[i];
        for (int j = 0; j < s[i]; j++) {
            cin >> v[i][j] >> w[i][j];
        }
    }
    for (int i = 1; i <= n; i++) {
        for (int j = m; j >= 0; j--) {
            for (int k = 0; k < s[i]; k++) {
                if (v[i][k] <= j) f[j] = max(f[j], f[j - v[i][k]] + w[i][k]);
            }
        }
    }
    cout << f[m] << endl;
    return 0;
}
