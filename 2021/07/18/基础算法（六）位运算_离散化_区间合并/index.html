基础算法（六）位运算

一、求整数 nnn 的二进制表示中第 kkk 位数

思路：

  1. 把第 kkk 位移到最后一位，使用 >> 运算符 n >> k
  2. 看个位是几，取出个位： x & 1

则公式为： n >> k & 1

二、\texttt{lowbit} 操作

lowbit(x)lowbit(x)lowbit(x) 为返回 xxx 的二进制数中最后一位 111 及其之后的部分。

例如：

x=5=(1010)_2\\ \Rightarrow lowbit(x)=(10)_2

返回最后一位 111 的位置公式： x & -x

推导：设 x=(1010⋯10⋯0)2x=(1010\cdots10\cdots0)_2x=(1010⋯10⋯0)​2​​ ，则 ∼x=(0101⋯01⋯1)2\sim x=(0101\cdots01\cdots1)_2∼x=(0101⋯01⋯1)​2​​

-x=\sim x + 1\\ x\&(-x)=x\&(\sim x+1)=\\ \begin{align} &1010\cdots100\cdots0\\ \&&0101\cdots100\cdots0\\ =&0000\cdots100\cdots0 \end{align}

应用：统计数 xxx 中 111 的个数

思路：应用 lowbit(x)lowbit(x)lowbit(x) 求出最后一个 111 的位置（二进制），之后去掉这个 111 及其之后的部分，即是减去当前位置的这个 111 及其后面的 000 转换成二进制之后的数 ，也就是减去 lowbit(x)lowbit(x)lowbit(x) 。 循环上述操作，直至 xxx 减为 000 。

int lowbit(int x) {
    return x & -x;
}

基础算法（七）离散化

整数有序、保序离散化

假定有序列 {an}\{a_n\}{a​n​​} ，其中对于 ∀ai,ai∈[0,109]\forall a_i,a_i\in[0, 10^9]∀a​i​​,a​i​​∈[0,10​9​​] ，值域的跨度非常大，但实际上 nnn 的值很小，序列非常稀疏。我们之后要使用这些数作为下标。如果为这么一些数去开辟一个 10910^910​9​​ 大小的数组很不值当，这时就要将这些数和自然数序列构建双射。这个过程就被称之为整数的离散化。

  • 序列中可能有重复元素：去重

    手写被排序之后调用的 unique() ：双指针算法

    C++

    vector<int>::iterator unique(vector<int> &a) {
    int j = 0;
    for (int i = 0; i < a.size(); i++) {
        if (!i || a[i] != a[i - 1]) a[j++] = a[i];
    }
    return a.begin() + j;
}

    Python

    def unique(a: list) -> int:
    j = 0
    for i in range(0, len(a)):
        if not i or a[i] != a[i - 1]:
            a[j] = a[i]
            j += 1
    return j

    int l = 0, r = alls.size() - 1;

基础算法（八）区间合并

给一系列区间，将区间中有交集的区间进行合并，得到并集。区间合并算法即快速的将两个存在交集的区间合并得到其并集。规定两个区间端点相同也可以合并。

步骤：

  1. 按照所有区间的左端点排序。需注意，已经按左端点排好序了，则之后不会再出现 li>li+k(k>0)l_i>l_{i+k}(k>0)l​i​​>l​i+k​​(k>0) 的情况

  2. 对于当前维护区间 [s,e][s, e][s,e] ，第 iii 个区间 [li,ri][l_i, r_i][l​i​​,r​i​​] 而言有如下三种情况：

    1. 当 s≥li,e≤ris\geq l_i,e\leq r_is≥l​i​​,e≤r​i​​ 时，被查看区间 [li,ri][l_i,r_i][l​i​​,r​i​​] 正处于维护区间内部，则该区间被维护区间吸收，当前维护区间不发生变化。
    2. 当 s≥li,e>ris\geq l_i,e>r_is≥l​i​​,e>r​i​​ 时，则被查看区间的右边界处于当前维护区间外侧，且有公共部分即存在交集。则需要更新当前维护区间的右边界为 rir_ir​i​​ ，即 e=rie=r_ie=r​i​​
    3. 当 s<e<li<ris<e<l_i<r_is<e<l​i​​<r​i​​ 时，被查看边界已经完全地处于当前维护边界之外了。由于左边界升序排列，那么当前这一区间 [s,e][s,e][s,e] 已经不可能和后续区间产生任何交集，不可能进行任何操作了。所以也就得到了一个答案，答案数加一，当前维护的区间焦点转向了区间 [li,ri][l_i,r_i][l​i​​,r​i​​]

    总结如下：

    \text{Check intersection and union: }[s,e],[l_i,r_i]\\ [s,e]\Rightarrow \begin{cases} [s,e]&s\geq l_i,e\leq r_i\\ [s,r_i]&s\geq l_i,e>r_i\\ [l_i,r_i],res=[s,e]&s

#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

vector<pair<int, int>> segs, res;

int main() {
    int n;
    scanf("%d", &n);
    for (int i = 0; i < n; i++) {
        int l, r;
        scanf("%d%d", &l, &r);
        segs.push_back({l, r});
    }
    sort(segs.begin(), segs.end());
    int s = segs[0].first, e = segs[0].first;
    for (auto &seg: segs) {
        if (e < seg.first) {
            if (e != segs[0].first) res.push_back({s, e});
            s = seg.first, e = seg.second;
        }
        else e = max(e, seg.second);
    }
    res.push_back({s, e});
    printf("%d", res.size());
    return 0;
}
n = int(input())
segs = [list(map(int, input().split())) for _ in range(n)]
segs.sort()
res = []
s = segs[0][0]
e = segs[0][0]
for seg in segs:
    if e < seg[0]:
        if e != segs[0][0]:
            res.append([s, e])
        s = seg[0]
        e = seg[1]
    else:
        e = max(e, seg[1])
res.append([s, e])
print(len(res))
