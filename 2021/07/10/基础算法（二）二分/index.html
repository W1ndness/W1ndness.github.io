前言

本文将介绍二分算法的相关内容及其模板，包括整数二分及浮点数二分的相关内容。

基础算法（二）二分

有单调性一定可以二分，但二分不需要单调性。二分的本质不是单调性，二分的本质是边界。

一、整数二分

给定一个区间 [l,r][l,r][l,r] ，在该区间定义了 ”由于“ xxx 的性质 \cal A 、\cal B ，在区间上的一点 xxx （即分划边界）会将区间分为了两部分 [l,x][l,x][l,x] 、 [x,r][x,r][x,r] ，使得区间 [l,x][l,x][l,x] 满足性质 \cal B ，而区间 [x+1,r][x+1,r][x+1,r] 满足性质 \cal A 。则点 xxx 使得对于区间 [l,x][l,x][l,x] 不满足性质 \cal A ，而对于区间 [x+1,r][x+1,r][x+1,r] 满足性质 AAA ，对于性质 \cal B 亦然。则区间可以一分为二，对于一种性质而言，一个区间满足，一个区间不满足，则可以找到边界点进行二分。

寻找 xxx 的方法：

有函数 check() 是检查元素满足什么性质

  • check() 检查 midmidmid 是否满足 [l,x][l,x][l,x] 上的性质 \cal B

    取 mid=⌊l+r+12⌋mid=\displaystyle\lfloor\frac{l+r+1}2\rfloormid=⌊​2​​l+r+1​​⌋ ， $\cal $如果满足，则更新区间为 [mid,r][mid,r][mid,r] ，即 l=midl=midl=mid ，因为 xxx 应该在 midmidmid 右侧的区间上；否则，如果不满足，则更新区间为 [l,mid−1][l,mid-1][l,mid−1] ，即 r=mid−1r=mid-1r=mid−1 ，因为 xxx 应该在 midmidmid 左侧的区间上。

  • check() 检查 midmidmid 是否满足 [x+1,r][x+1,r][x+1,r] 上的性质 \cal A

    取 mid=⌊l+r2⌋mid=\displaystyle\lfloor\frac{l+r}2\rfloormid=⌊​2​​l+r​​⌋ ，检查 midmidmid 是否满足 [mid,r][mid,r][mid,r] 上的条件或性质，如果满足，则更新区间为 [l,mid][l,mid][l,mid] ，即 r=midr=midr=mid ；否则，如果不满足，则更新区间为 [mid+1,r][mid+1,r][mid+1,r] ，即 l=mid+1l=mid+1l=mid+1 。

    步骤：

  1. 写出 check() 函数，默认二分是取 mid=⌊l+r2⌋mid=\displaystyle\lfloor\frac{l+r}2\rfloormid=⌊​2​​l+r​​⌋
  2. 看 call 函数的时候条件为 true 应该如何更新区间，即 midmidmid 应该赋给 lll 还是 rrr
  3. 若是 l=midl=midl=mid 则取 midmidmid 的条件改为 mid=⌊l+r+12⌋mid=\displaystyle\lfloor\frac{l+r+1}2\rfloormid=⌊​2​​l+r+1​​⌋ ；若是 r=midr=midr=mid 则不变

为什么要改 midmidmid 为加一？

由于除法是向下取整，当需要停止二分前，两侧区间满足条件 l=r−1l=r-1l=r−1 。midmidmid 此时取值是 lll 。这时再进行 check(mid) ，若结果为 true，则赋值句就变成了 l=ll=ll=l ，区间会永远的卡死在当前 [l,r][l,r][l,r] 内死循环。如果该分子为加一， midmidmid 即等于 rrr ，区间变为 [r,r][r,r][r,r] ，二分停止。

模板：假设 check() 函数已经定义 bool check(int);

int bsearch(int l, int r) {
  	while (l < r) {
    	int mid = l + r >> 1;
    	if (check(mid)) r = mid;
    	else l = mid + 1;
  	}
  	return l;
}

    	if (check(mid)) l = mid;
    	else r = mid - 1;
  	}

思想：更新区间，保证 xxx 一定被区间覆盖，不断缩小区间，直到区间缩小为1，即得到了 xxx 。无解就是最后的答案是小于等于 xxx 的，二分不会无解，无解是因为题目中的条件而无解，二分总是用性质判断会找到结果。

二分的几个小问题：

  • 性质的选取？ check() 该判断什么？

    性质的选取实际上是任意的，只要是一个能够卡出边界的性质就可以了。但一定要注意更新区间的时候你设置的性质里面到底包不包含 midmidmid 指向的元素，这决定了更新区间时候 111 的细微差别。

  • midmidmid 是否要 +1+1+1 ？

计算 midmidmid 到底要不要加一主要是看更新区间时语句 ?=mid?=mid?=mid 中 ??? 到底是谁。如果是 l=midl=midl=mid 就需要改一下 midmidmid 的赋值语句了。这么做的目的就是防止区间更新之后还是原来的区间，最后导致了死循环。相关推导上面已经给出了。

二、浮点数二分

浮点数二分不存在像整数二分的边界问题，答案在左边就更新区间为左区间，答案在右边就更新区间为右区间。二分停止的条件是 r−lr-lr−l 达到了一个阈值 δ\deltaδ ，即 r−l≤δr-l\leq\deltar−l≤δ 时二分停止得到答案。

模板：

double bsearch(int l, int r, int delta) {
  	while (r - l > delta) {
		double mid = (l + r) / 2;
    	if (check(mid)) r = mid;
    	else l = mid;
  	}
  	return l;
}

AcWing 789

AcWing 789

找边界！！找边界！！边界将序列分为两部分，顾想到二分。

依题意，要找到被查询元素的上界和下界。首先看上界。上界的意味是第一个 ≥x\geq x≥x 的数，这时就可以定义 check() 的性质是 q[mid] >= x ，当然这里定义为 q[mid] < x 也未尝不可，只不过变成了反方向的寻找，找到的是上界的前一个数，更新区间的时候 lll 应该被更新成 mid+1mid+1mid+1 。之后依据区间的变化和 xxx 被囊括在哪一个区间内去更新区间和判断要不要改动 midmidmid 的取值即可。再看下界，下界的意味是最后一个 ≤x\leq x≤x 的数，则思路和上面一样，可以找到上下界。本题结束。

#include <iostream>
using namespace std;

const int N = 100010;

int q[N];

int main() {
    int n, k;
    scanf("%d%d", &n, &k);
    for (int i = 0; i < n; i++) scanf("%d", &q[i]);
    while (k--) {
        int x;
        scanf("%d", &x);
        int l = 0, r = n - 1;
        while (l < r) {
            int mid = l + r >> 1;
            if (q[mid] >= x) r = mid;
            else l = mid + 1;
        }
      	/* or write as following
      	while (l < r) {
      		int mid = l + r >> 1;
      		if (q[mid] < x) l = mid + 1;
      		else r = mid;
      	}
      	*/
        if (q[l] != x) printf("-1 -1\n");
        else {
            printf("%d ", l);
            l = 0, r = n - 1;
            while (l < r) {
                int mid = l + r + 1 >> 1;
                if (q[mid] <= x) l = mid;
                else r = mid - 1;
            }
            printf("%d\n", l);
        }
    }
    return 0;
}

#include <iostream>
#include <cmath>
using namespace std;

const double eps = 1e-8;

int main() {
    double n;
    cin >> n;
    double l = -10000, r = 10000;
    while (r - l > eps) {
        double mid = (l + r) / 2;
        if (mid * mid * mid >= n) r = mid;
        else l = mid;
    }
    printf("%.6f", l);
    return 0;
}
